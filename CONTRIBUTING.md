# Contributing

Спасибо за интерес к проекту. Вот как можно помочь.

---

## Что особенно нужно

### 1. Новые opportunity patterns
Файл `skill/references/opportunity-patterns.md` содержит паттерны для поиска возможностей в vault. Если вы нашли в своих заметках паттерн, который привёл к реальному результату (статье, продукту, автоматизации) — это ценный вклад.

Формат:
```markdown
### Pattern N: Название
**Сигнал в vault:** что именно вы увидели в заметках
**Тип возможности:** статья / продукт / автоматизация
**Пример:** конкретный случай (можно анонимно)
```

### 2. Vault структуры для конкретных ниш
Текущие структуры в `skill/references/vault-structures.md` — универсальные. Нужны структуры под конкретные профили: AI educator, automation builder, indie hacker, AI researcher.

### 3. Реальные примеры анализов
Анонимизированные примеры Vault Intelligence анализов (с вашего разрешения) помогают другим понять, что ожидать. Добавляйте в `examples/`.

### 4. Переводы
README.md на английском (для international community).

## Как сделать PR

1. Fork репозитория
2. Создайте ветку: `git checkout -b feature/my-improvement`
3. Внесите изменения
4. Проверьте, что markdown файлы открываются корректно в Obsidian
5. PR с описанием: что изменили и почему

---

## Принципы, которым должны соответствовать изменения

- **Zettelkasten-совместимость**: изменения не должны противоречить принципам Луманна / Аренса
- **AI Creator фокус**: таксономии и паттерны должны быть релевантны для AI-энтузиастов
- **Actionable over theoretical**: каждое добавление должно давать практический результат
- **Примеры обязательны**: любой новый паттерн или шаблон — с примером использования
